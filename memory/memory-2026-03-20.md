# Session — 2026-03-20

## Decisions Made

### App Structure
- **www.bub.ai** — TanStack Start, marketing/landing pages
- **app.bub.ai** — TanStack Start w/ TanStack Query, dashboard + chat + agent management (consolidated from originally having a separate chat.bub.ai Vite SPA)
- Decided to merge chat into app.bub.ai to avoid cross-domain auth complexity

### Auth
- JWT + Web Crypto API + HttpOnly cookies on `.bub.ai`
- Custom auth, no third-party provider (no Clerk, no Auth.js)

### LLM Provider — BYOK (Bring Your Own Key)
- Users paste in their own API keys
- Supported providers: Claude (API key or OAuth), Ollama (local)
- User picks their model, app is provider-agnostic
- Keys encrypted at rest with Web Crypto AES-GCM

### Runtime — Cloudflare Workers
- www → Cloudflare Pages
- app → Cloudflare Pages w/ Workers functions
- API → Cloudflare Workers
- DB → D1 or Turso (undecided)
- KV → Cloudflare KV for sessions/cache
- Queues → Agent task dispatch
- Durable Objects → Long-running agent state / chat sessions

## Core Product Vision
User logs in → configures LLM provider + key → chats with AI agents → agents spin up and modify other AI agents to complete tasks like creating and modifying apps.

### Layout Architecture (Key Decision)
- When **signed out**: Header + main content + Footer (traditional layout)
- When **signed in**: Full-screen AuthChatLayout — Chat sidebar on left, Content iframe on right, NO header/footer
- Content iframe loads routes like `/projects`, `/settings` which use AuthContentLayout (white bg)
- AccountDropdown in Chat sidebar controls iframe navigation via `onNavigate` prop
- Iframe src persisted via URL hash (e.g. `/#/settings`) — survives hard refresh
- Content iframe only renders after mount to avoid SSR hydration mismatch
- Sign in and create account both navigate to `/` (which shows AuthChatLayout when authed)

- `UserPromptSubmit` hook in `.claude/settings.local.json`
- Checks `app-bub/errors/` for `.md` files on every user message

- No latency cost when no errors exist (outputs nothing)
- CLAUDE.md instructs to fix errors when "BROWSER ERRORS DETECTED" appears in context
